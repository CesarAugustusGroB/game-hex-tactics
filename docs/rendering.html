Dos sistemas de coordenadas

Axial hex (q, r)

La rejilla lógica (flat-top, size = 40). Convertir con HexUtils.hexToPixel / pixelToHex. La clave canónica de Map es HexUtils.key({q,r}) — nunca stringificar a mano.

PIXI world

El contenedor worldRef se panea/zoomea. Para hacer lookups de hex desde pantalla: world.toLocal(...) primero (screen → world), luego pixelToHex.

Pipeline de terreno (drawTerrain.ts) — dos pasadas

Pass 1 — terrainGfx (Graphics por hex)

Polígono superior (color plano o Texture para RIVER/SAND/SEA/DEEP_SEA) + muros laterales S/SE/SW sombreados. Los muros se dibujan solo para biomas no-texturizados (SAND, ROCKY). Los hex iteran en height ascendente, así que el terreno más alto se dibuja después (encima) de sus vecinos bajos.

Pass 2 — terrainOverlayRef (TilingSprite + máscara)

GRASSLAND / FOREST / HILL / MOUNTAIN / SNOW reciben una textura tileada en world-space recortada por máscara. La máscara base = unión de los tops del bioma + la cara de cliff visible contra cada vecino más bajo (un quad de alto (hexH − nH) en el borde S/SE/SW). Resultado: la textura pinta de forma continua sobre top y pendiente, sin muro oscuro ni invasión del vecino.

El "dive" estratégica ↔ táctica

click hex (isScanning)→ captura coords de noise→ resolution ×4.5→ viewMode = TACTICAL

El mismo generateWorldData() produce ambas vistas. Bucear captura las coordenadas en espacio-noise del hex clicado para samplear ese parche, y multiplica la resolución (divisor menor = más detalle). "Return to strategic" resetea noiseOffset y resolution.

Gotchas de rendimiento (PIXI v8)

Un Filter se crea una vez y se reutiliza. Recrearlo por draw recompila un GlProgram y filtra (leak) el anterior — Container.destroy() no libera sus filtros en v8. Los dos filtros de agua se cachean de por vida en ctx.waterFilters (por tipo) y se reasignan a cada contenedor recién construido.

Color.multiply(number) trata el número como hex int vía bit-shifts (0.7 | 0 === 0 → negro). Para sombrear, pasa siempre un array RGB normalizado [s, s, s, 1].

Capas que viven en el app, no por-frame: filtros y texturas horneadas se crean una vez; solo los display objects por-frame son descartables. La rejilla se redibuja en su propio gridGfx desde useEffect([gridData, showGrid]) para que el toggle GRID no reconstruya todo el pipeline (que recompila shaders de agua).

Estado espejado en refs

isScanningRef y noiseOffsetRef se sincronizan vía useEffect porque el handler pointertap se registra una vez al montar y cerraría sobre estado obsoleto. Regla: todo estado nuevo leído dentro de handlers PIXI de larga vida debe espejarse igual — no re-registrar handlers por render.
