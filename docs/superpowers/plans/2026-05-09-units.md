# Units MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a click-to-place units layer with `STRATEGIC` army icons and `TACTICAL` unit icons, per `docs/superpowers/specs/2026-05-09-units-design.md`.

**Architecture:** All logic lives in `src/components/GameCanvas.tsx`. New React state (`armies`, `currentStrategicHex`, `isPlacing`) plus mirror refs for use inside long-lived PIXI handlers. A `unitsGfx: PIXI.Container` rebuilt on relevant changes (analogous to `terrainGfx`). Two SVG assets in `public/units/` loaded once via `PIXI.Assets`.

**Tech Stack:** React 19, PIXI.js 8, simplex-noise 4, TypeScript 5, Vite 8.

**Verification model (deviation from default TDD):** This repo has no test runner. Each task verifies via:
- `npm run build` — `tsc -b` + `vite build` (type-check is the contract).
- `npm run lint` — ESLint flat config.
- Manual browser flow at http://localhost:5174/ (port 5174 because port 5173 is already used by the parent worktree's dev server).

## Task 8: TACTICAL unit-icon render

**Files:**
- Modify: `src/components/GameCanvas.tsx`

- [ ] **Step 1: Extend `drawUnits` with the TACTICAL branch**

Modify `drawUnits` so the function body becomes:

```tsx
const drawUnits = useCallback(() => {
  const c = unitsGfx.current;
  c.removeChildren();
  const armyTex = armyTextureRef.current;
  const unitTex = unitTextureRef.current;
  if (!armyTex || !unitTex) return;

  if (viewMode === 'STRATEGIC') {
    armies.forEach((_units, key) => {
      const strategicHex = HexUtils.fromKey(key);
      const tile = gridData.find(d => d.hex.q === strategicHex.q && d.hex.r === strategicHex.r);
      if (!tile) return;
      const pos = HexUtils.hexToPixel(strategicHex);
      const sprite = new PIXI.Sprite(armyTex);
      sprite.anchor.set(0.5, 1);
      sprite.x = pos.x;
      sprite.y = pos.y - TERRAINS[tile.type].height - 6;
      sprite.width = 40;
      sprite.height = 40;
      c.addChild(sprite);
    });
    return;
  }

  // TACTICAL
  if (!currentStrategicHex) return;
  const units = armies.get(HexUtils.key(currentStrategicHex)) ?? [];
  units.forEach(u => {
    const tile = gridData.find(d => d.hex.q === u.tacticalHex.q && d.hex.r === u.tacticalHex.r);
    if (!tile) return;
    const pos = HexUtils.hexToPixel(u.tacticalHex);
    const sprite = new PIXI.Sprite(unitTex);
    sprite.anchor.set(0.5, 1);
    sprite.x = pos.x;
    sprite.y = pos.y - TERRAINS[tile.type].height - 4;
    sprite.width = 32;
    sprite.height = 32;
    c.addChild(sprite);
  });
}, [armies, viewMode, gridData, currentStrategicHex]);
```

The dependency array now includes `currentStrategicHex` so re-diving causes a redraw.
