Solo el equipo blue tiene IA. Snapshot del comportamiento autorado en src/data/ai.json + src/battle/ai/. Cada tick: percibe → calcula peligro → despliega (amass) → decide por grupo (abajo).

Contraataque · cuánto peligro baja el umbral de lanzamiento

// VP: cercanía del rival a ganar + mi déficit vpDanger = 0.5·(enemyScore/200) + 0.5·(max(0, enemyScore−myScore)/200) // presión observable pressure = 0.34·breachers + 0.17·raiders + 0.30·(rival holdea centro) danger = clamp(0.6·vpDanger + 0.4·pressure, 0,1)
// umbral de unidades para lanzar el frente launchShare = max(4, ⌈ bandShare · (1 − 0.75·danger) ⌉ ) // danger=0 → launchShare = bandShare (espera lleno) // danger=1 → launchShare ≈ 25% (contraataca ya) frontReady = toda banda ≥ launchShare o ya no puede crecer

Progreso por tiers

Tier 0 · Percepción✓ HECHO

"Los ojos": perceive() puro. Breachers/raiders, control del centro, centroide enemigo. Cimiento de todo lo reactivo.

Tier 1 · Defensa reserva✓ HECHO

La reserva intercepta raids a mi zona: march a la brecha → hold. Cierra el exploit de farmear mi retaguardia.

Contraataque✓ HECHO

danger (VP + presión) baja el umbral: si voy perdiendo, lanzo el frente con menos unidades amasadas.

Tier 2 · Combate✓ HECHO

Cavalry carga, skirmisher unleash (kite), infantry holdea el centro. Cada tipo pelea con su asimetría.

Tier 3 · TerrenoPENDIENTE

Rutar por moveCost bajo, evitar RIVER/SNOW, buscar altura para el bonus de daño y para holdear.

Tier 4 · Foco de fuegoPENDIENTE

Concentrar en el clúster enemigo más débil/cercano en vez de apuntar siempre al centro abstracto.

Tier 5 · Estrategia VPPENDIENTE

Jugar según el marcador: holdear si gano, abrir un 2º eje de raids si pierdo.

Retirada / banishPENDIENTE

Retirar grupos heridos y banish-con-reembolso (requiere nueva capacidad en AiTickState).
